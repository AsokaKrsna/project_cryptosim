Documentation

Everything you need to go from zero to wiring cryptographic circuits in minutes.

What is CryptoSIM?

CryptoSIM is an offline-first native desktop application built with Tauri and Rust. It gives you a visual, node-based canvas where you wire together 310 cryptographic and blockchain primitives — called Bricks — without writing a single line of code. Data flows through typed wires in real time as you build.

Who is it for?

  • Students — build intuition for SHA-256, AES, ECDSA, and ZK proofs by seeing the bytes flow
  • Educators — create interactive lessons with locked bricks and guided objectives using Professor Mode
  • Developers — prototype key derivation pipelines, signing flows, or Merkle proofs visually before writing code
  • Researchers — simulate attack scenarios (51% attack, padding oracle, nonce reuse) in a safe sandbox

Installation

Windows (Recommended)

Download CryptoSIM_Setup_v1.0.2.exe and run it. The NSIS wizard will install CryptoSIM to C:\Program Files\CryptoSIM\, create a Start Menu entry, and register .cryptosim as a file type so you can open canvas files by double-clicking them.

Windows SmartScreen Warning

On first launch, Windows may show "Windows protected your PC." Click More info → Run anyway. This warning appears because the binary is not yet code-signed (an EV certificate costs ~$300/year). It does not affect functionality and diminishes as the app gains reputation.

Linux (Debian/Ubuntu)

Download the .deb package and install:

sudo dpkg -i CryptoSIM_v1.0.2_amd64.deb
sudo apt-get install -f   # resolve any missing dependencies
macOS

macOS support requires an Apple Developer account to sign and notarize the binary. We are currently seeking donations to fund this. If you need macOS support urgently, get in touch.

System Requirements

  • Windows 10 or 11 (64-bit) — WebView2 required (pre-installed on Win11; auto-installs on Win10)
  • Linux — any modern distribution with WebKit2GTK 4.1
  • ~3 MB disk space; no internet connection required after install

The Bricks System

Every operation in CryptoSIM is a Brick — a self-contained computation node with typed input and output ports. Building a "circuit" means dragging bricks from the palette onto the canvas and connecting their ports with wires.

Three categories of bricks

  • Source bricks — produce data with no inputs: STRING, NUMBER, RANDOM BYTES, TIMESTAMP.
  • Transform bricks — consume inputs and emit outputs: e.g. SHA-256, AES ROUND, ECDSA, MINER. These make up the bulk of the 310 bricks.
  • Sink bricks — display or consume the final result: DISPLAY, LOG, INSPECTOR.

How bricks compute

  1. You connect an upstream brick's output port to a downstream brick's input port.
  2. The execution engine immediately recomputes the downstream brick using topological ordering — so every dependent brick refreshes automatically.
  3. The result flows along the wire as a live value, color-coded and visible on hover.
  4. If a brick gets an error (wrong type, missing input, cryptographic failure) it shows a red status and a specific message — not a silent failure.

Brick palette

All bricks are organized by cluster in the left-hand palette. Click a cluster header to expand it, then drag a brick onto the canvas. You can also search by name using the palette search bar. Hover over any brick in the palette to see its port signature before placing it.

Pro tip — Display Modes

Each brick has multiple display modes accessible via the mode button in its header: standard (default), compact (small card), and expanded (shows visualization panel with step-through controls for complex bricks like AES Round or ECC Scalar Mult).

Port Types & Colors

Every port has a strict type. Wires can only connect ports of compatible types. This prevents invalid cryptographic operations — you cannot accidentally feed a BigInt where raw bytes are expected.

Bytes Raw byte arrays. SHA digests, AES ciphertext, keys, signatures, nonces.
Number Arbitrary-precision BigInt. RSA moduli, scalar values, iteration counts.
Point An (x, y) coordinate on an elliptic curve. Used by ECC bricks.
Boolean True / False. Output of signature verifiers and hash comparators.
Field Element A value in a finite field — used by advanced ZK and pairing bricks.
Array Ordered list of typed values. Merkle tree leaf lists, key arrays.
BitString A string of '0'/'1' characters — used by logic and bitwise bricks.
Any Passes through whatever it receives. Used by utility and display bricks.

When you hover over a port, a tooltip shows the type name and a short description. When you drag a wire from an output, all incompatible input ports on other bricks dim — making valid connections obvious at a glance.

Type Coercion

CryptoSIM does not silently coerce types. If you need to convert a hex string to raw bytes, use the DECODE brick (Cluster 02). If you need to turn bytes into a BigInt, use the TYPE CAST brick.

Execution Engine

CryptoSIM does not use a typical event loop. It uses a topological execution engine. When you change a value, the engine maps the entire graph, sorts it from left to right (dependencies first), and fires the compute() method on every affected brick in order.

This guarantees that by the time a signature verifier runs, both the message hash and the public key have finished computing.

Run All (Ctrl+R)

By default, CryptoSIM evaluates the graph automatically as you type or connect wires. However, if you load a large file (50+ bricks) or experience visual desync, click the ▶ Run All button in the top menu or press Ctrl+R. This forces a complete re-evaluation of every brick on the canvas.

Background threads

Most operations (SHA-256, AES) are so fast they execute synchronously inside the React render cycle via WebAssembly or Rust bindings. Heavy workload bricks (like PBKDF2 with 10M iterations, or the Proof-of-Work Miner) offload explicitly to Web Workers, freeing up the UI thread so you can continue panning the canvas.

Step-by-Step Execution Debugger

By default, CryptoSIM computes instantly. But for educational purposes (like YouTube videos or classroom demos), you can slow it down.

  1. Look at the top toolbar next to Run All. Change the speed dropdown from ⚡ Instant to 🐢 0.5s or 🐌 1.0s.
  2. Click Run All. A floating glassmorphic toolbar will appear at the bottom of the screen.
  3. Watch as the execution engine pulses each node green in topological order!

Use the ⏸ Pause, ⏪ Step Back, and ⏩ Step Forward buttons on the toolbar to manually trace the data flow through complex circuits.

Deep Data Inspector

Ever wonder exactly what those raw bytes look like inside a SHA-256 hash or an AES ciphertext?

Interactive Ports

Any time you see a byte array value in the right-hand Inspector Panel, click it! This opens the Deep Data Inspector Modal, providing a professional-grade hex dump view, ASCII representation, and Base64 output.

Auto-Layout & Clipboard

  • Magic Wand: Click the ✨ icon in the top toolbar. The Dagre layout engine will instantly untangle your wires and organize your circuit into a clean left-to-right hierarchy.
  • Copy / Paste: Select multiple bricks and press Ctrl+C. Press Ctrl+V anywhere on the canvas to clone them. Wires between copied bricks are perfectly preserved!

Brick Clusters

The 310 bricks are organized into 17 distinct clusters. Here is a high-level overview of what you can find in each:

01 I/O & Data Types

Input sources (Text, Hex, Numbers, Files) and display sinks (Logger, Inspector, Boolean LED).

STRING INPUT · RANDOM BYTES · DISPLAY
02 Encoding & Formats

Convert data between Hex, Base64, Base58, UTF-8, and format structures like PEM and DER.

HEX DECODE · BASE58 ENCODE · PEM BUILDER
03 Logic & Bitwise

Gates (AND, OR, XOR), shifts, rotations, and multiplexers used in low-level cryptographic algorithm design.

XOR · BIT ROTATE RIGHT · MULTIPLEXER
04 Mathematics

BigInt arithmetic, modular arithmetic, prime generation, and finite field operations.

MODULAR ADD · MODULAR INVERSE · RAND PRIME
05 Symmetric Crypto

Block ciphers, stream ciphers, modes of operation, and padding schemes.

AES ENCRYPT · CHACHA20 · PKCS#7 PAD
06 Hashing

Cryptographic hash functions spanning the SHA-1, SHA-2, SHA-3, and BLAKE families.

SHA-256 · KECCAK-256 · BLAKE3 · RIPEMD-160
07 MAC & KDF

Message authentication codes and Key Derivation Functions (password hashing).

HMAC-SHA256 · PBKDF2 · SCRYPT · ARGON2
08 Asymmetric & ZK

Public-key cryptography (RSA, ECC), signatures (ECDSA, Schnorr), and zero-knowledge proofs.

GEN KEYPAIR · ECDSA SIGN · GROTH16 PROVER
12 Consensus

Mechanisms for distributed agreement, including Proof-of-Work mining loops and difficulty targets.

MINER · VERIFY POW · DIFFICULTY TARGET
16 Attack Simulations

Simulate common cryptographic vulnerabilities like nonce reuse or length extension attacks.

NONCE REUSE EXPLOIT · PADDING ORACLE

See the Brick Palette in the app for the full list of all 310 bricks.

Professor Mode EDUCATORS

Professor Mode transforms CryptoSIM from an open sandbox into an interactive lesson builder. You can author a cryptographic challenge, lock down the solution, and send it to your students as a single file.

How to create a lesson

  1. Build the circuit: Lay out the full working solution on the canvas (e.g. a SHA-256 pre-image challenge).
  2. Enable Professor Mode: Click the "Professor Mode" button in the top right header. A new toolbar will appear at the bottom of the screen.
  3. Set the Objective: Click "Edit Objective" in the toolbar. Write a markdown description (e.g., "Find the nonce that results in a hash starting with two zeros") and define the success condition by selecting the output port of your verification brick.
  4. Lock and Hide Bricks: Hover over any brick on the canvas. Use the 🔒 Lock button to freeze the brick (students cannot delete it or move it). Use the 👁 Hide button to make the brick invisible to students (useful for hiding the answer checker).
  5. Export: Click "Export Lesson" in the top bar. This generates a .cryptosim file that contains your locked state.
Student Experience

When a student opens your exported .cryptosim file, the app automatically switches into "Lesson Active" mode. They will see the objective panel, but they will not have access to the Professor Toolbar. They cannot unlock bricks or reveal hidden ones.

Files & Sharing

The .cryptosim Format

CryptoSIM saves all canvas state — bricks, wires, positions, and professor mode constraints — into a single lightweight JSON file with the .cryptosim extension. Since all computation happens locally, you can share these files via email, Slack, or Discord without worrying about API keys or cloud databases.

A typical 10-brick canvas results in a file size under 5 KB.

Autosave

CryptoSIM automatically saves your canvas to your browser's (or Tauri's) IndexedDB every 1.5 seconds. If you accidentally close the app or experience a crash, simply reopening the app will instantly restore your exact state.

FAQ

Is CryptoSIM open source?

No, CryptoSIM is distributed under a Custom Proprietary Freeware license. It is 100% free to use for personal, educational, and commercial purposes, but the source code is not published or redistributable. You own whatever .cryptosim files you create.

Does it work entirely offline?

Yes. The only time CryptoSIM connects to the internet is if you click a link to this website or the donation page. All cryptographic execution, including ZK proof generation, happens locally on your machine.

Why is there no macOS version?

macOS Gatekeeper makes it exceptionally difficult to distribute an unsigned binary. Signing and notarizing requires a paid Apple Developer License ($99/yr), which the project does not currently have. We are collecting donations to fund community macOS distribution.

Troubleshooting

Canvas is blank or shows an error

If a malformed .cryptosim file or an experimental brick crashes the React renderer, an Error Boundary will appear showing a red crash screen with a stack trace. Click the 🗑 Clear Canvas & Reload button to purge the corrupted autosave state and start fresh.

A wire is connected but the value isn't updating

Click Ctrl+R (Run All) to force the execution engine to recalculate the entire topological graph. This usually only happens if you manually intervene in Professor Mode locking while data is mid-flight.

"Unknown brick" when loading a file

This means the .cryptosim file was created in a newer version of CryptoSIM that has bricks your current version doesn't recognize. Download the latest installer from the website and try again.
